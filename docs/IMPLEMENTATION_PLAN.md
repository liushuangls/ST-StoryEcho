# StoryEcho 实现计划

## 阶段0：项目基础

- TypeScript 构建、扩展 manifest、设置仓库、日志、错误边界和单元测试；
- 可通过 Git 地址直接安装的 `dist/` 产物；
- 基础设置面板和响应式布局。

状态：已完成。

## 阶段1：LLM Provider

- 主连接 Provider；
- 自定义 OpenAI 兼容 Provider；
- Base URL 规范化；
- 自定义请求经 SillyTavern 自带 Chat Completions 后端转发；
- Key 通过扩展设置持久化和同步；
- 超时、取消、结构化输出、连接测试和可选回退。

状态：已完成核心链路，等待更多真实接口样本。

## 阶段2：聊天记忆存储

- `StoryMemory` Schema 和 `chatMetadata` Repository；
- Schema 版本、聊天 UUID 和分支隔离；
- 来源哈希与失效检测；
- 记忆查看、编辑和删除。

状态：权威存储、分支隔离、保守失效重建和记忆编辑器已完成。

## 阶段3：Vector Storage 与自定义 Embedding

- Vector Storage insert/list/query/delete/purge；
- 哈希碰撞处理、增量同步和完整重建；
- Embedding 来源、端点和模型变化检测；
- 自定义 OpenAI 兼容 Embedding 自动使用 SillyTavern 内置 `/proxy/`；
- 火山方舟多模态 Embedding 独立适配、限并发请求与响应校验；
- 响应校验、连接测试、Key 设置同步；
- 预生成向量继续由 Vector Storage 保存与检索。

状态：核心链路和火山方舟真实接口适配已完成，继续扩展其他真实接口兼容测试。

## 阶段4：剧情抽取

- 固定轮次切块；
- 候选事件提取 Prompt 与 Schema；
- JSON 解析和修复重试；
- 手动抽取与窗口边界自动追赶；
- 后台抽取队列。

状态：同步安全补齐与AI回复后的有界单批后台队列均已完成。

## 阶段5：事件整理

- 相似旧事件预检索；
- CREATE/MERGE/UPDATE/RESOLVE/SUPERSEDE/IGNORE；
- 原子事务更新、向量同步队列和手工修改保护。

状态：核心链路已完成，等待真实聊天样本调优。

## 阶段6：滑动窗口与召回

- 按轮次/消息计算保留边界；
- 有覆盖游标的独立阶段总结条目；
- `W` 最小原文窗口、`N` 总结间隔与 `S` 总结窗口；
- 未总结原文与最小 W轮原文的并集保留；
- LLM 上下文查询改写、缓存与本地规则回退；
- 双路召回、轻量重排和 Token 预算；
- 阶段总结位于近期原文前、本轮 system 召回位于当前 User 前的 Prompt Interceptor；
- 请求级注入不修改聊天记录和当前用户原文；
- 后台调用与正常生成隔离。

状态：滑动窗口、分批阶段总结、结构化召回和请求级注入顺序已完成。

## 阶段7：检查器与稳定性

- 本次上下文检查器；
- 消息编辑、删除和 Swipe 失效；
- 分支索引复制/重建；
- 重建进度和取消；
- 导入导出验证、大聊天性能测试；
- 与世界书、MVU 和常见扩展兼容测试。

状态：基础统计、最近检查、调试报告，以及消息编辑、删除、Swipe 和分支失效重建已完成；其余持续开发。

## 阶段8：调度与结构化记忆管线改造

- 使用单一任务协调器串行化前台上下文准备、手动处理和后台维护；
- 后台运行期间到来的前台生成进入等待队列，当前批次结束后优先执行；
- 使用跨真实回复生命周期的前台租约，阻止回复生成期间启动后台 LLM；
- 以请求标记识别 StoryEcho 内部 `generateRaw`，不再用全局内部深度跳过并发用户生成；
- 结构化任务统一采用 JSON Object、JSON Schema、同 Provider 普通模式三级回退；
- 抽取失败时按完整轮次自适应拆批并有界退避；
- 使用剧情、状态、关系、承诺、揭示和线索六类输出结构；
- 剧情与冲突不再通用原子化，状态、关系和承诺按稳定逻辑槽归一化；
- 状态取代、承诺完成和明确否定由确定性规则优先处理；
- 阶段总结区分 UI Persona 与剧情用户角色并校验新增稳定身份事实。

状态：代码与自动化回归已完成；仍需安装最新构建后执行一轮真实页面回归。完整设计与验收标准见 [PIPELINE_V2.md](PIPELINE_V2.md)。

## 阶段9：证据权威与前台延迟加固

- 对抽取字段中的具体专名执行来源落地校验，并兼容此前已确认实体；
- Assistant推断本地降级，明确剧情推进继续保留；
- 阶段总结固定拆为确认、当前、未解、推测和失效五个权威区；
- 精确事实问答只使用confirmed记忆和总结中的确认/当前区；
- 结构化失败先做本地JSON语法修复；后台在模式降级边界为已排队前台生成让行；
- 旧版无来源专名记忆只读隔离，手工编辑保护不变。

状态：代码与自动化回归已完成，等待最新构建的真实页面回归。

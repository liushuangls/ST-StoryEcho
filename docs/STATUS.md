# 当前实现状态

更新时间：2026-07-22

## 0.20.4 长剧情连续性与事实校验

- 每条新阶段总结会附带紧邻上一条总结末尾最多 `5000` 字符作为连续性参考；本批原文保持较新事实最高优先级，旧总结不会被重写；
- 阶段总结按剧情复杂度采用软长度目标：常规约 `1000–1600` 个中文字符，多线复杂批次约 `2200` 字符，重复训练、照料与即时面板状态优先压缩；
- 全局骨架的首次生成、增量更新与重建在每批候选生成后增加完整正文事实一致性校验，重点校正实体境界、能力归属、物品、行动是否发生、关系、时间因果与信息确定程度；
- 阶段总结管理区新增真实注入状态标记，以及“重建全部阶段总结与骨架”：全部总结草稿成功后才原子替换旧集合，再从新总结干净重建骨架；任一总结批次失败保留旧集合；
- 提示词进一步把即时角色状态与 NPC 档案交给近期上下文、MVU变量和世界书，长期总结集中记录会继续影响剧情的事件、因果、成长、关系和伏笔。

## 0.20.3 阶段总结长批次与世界书背景

- 阶段总结单批原文目标上限由 `64000` 提升到 `100000` 字符，仍只在完整轮次边界收束；单个超限完整回合保持整体处理并写入调试轨迹；
- 阶段总结与全局骨架共用蓝灯/绿灯世界书构建逻辑：蓝灯常驻完整条目最多 `20000` 字符，本批来源直接命中的绿灯完整条目最多 `10000` 字符；
- 两组世界书预算不再受抽取参考 Token 上限二次压缩，不读取角色卡，也不把设定资料当成已发生剧情或当前状态；
- 调试轨迹新增阶段总结原文、最终请求、蓝灯/绿灯条目数与字符数及估算 Token，便于定位超长上下文。

## 0.20.2 题材自适应剧情整理

- 阶段总结、全局剧情骨架和剧情记忆抽取不再默认采用侦探/案件叙事；先识别原文题材，再按其世界规则与长期剧情重心分配篇幅；
- 修仙/玄幻剧情会优先保留境界突破、功法术法、体质灵根、修炼瓶颈、传承机缘、法宝丹药、宗门势力、师徒同伴关系和历练目标；恋爱、日常、冒险与权谋题材也采用各自的叙事重点；
- 阶段总结和全局骨架改为自然剧情纪要，可按时间、因果、人物成长、关系或势力线自由分段；不再要求、补齐、解析或校验固定标题，人工编辑也只校验非空与长度；
- 严格事实核验请求不注入自由格式总结或骨架，只使用confirmed结构化记忆、当前状态校正与近期原文，避免把自然叙事中的主张或推测误当成事实。

## 0.20.1 骨架正文默认折叠

- 全局剧情骨架卡片默认只显示状态、覆盖范围和更新时间；点击卡片标题后再展开正文、说明及保存/立即更新操作；收起不会丢失尚未保存的编辑。

## 0.20.0 全局剧情骨架

- 最近 `S` 条阶段总结之外的历史现在滚动合并为始终注入的全局剧情骨架，避免长聊继续推进后最早阶段完全离开请求；尚未成功并入骨架的归档总结继续直接注入，不会因等待、失败或前台让行出现信息空洞；
- 新聊天在第 `S + 1` 条有效阶段总结出现时以当时全部有效总结首次生成；已有长聊天加载稳定约 3 秒后自动补建；已有骨架在每条尚未覆盖的总结首次归档时立即更新，手动“处理窗口外历史”和“立即更新”可依次处理全部积压；
- 首次生成与手动重建按旧到新、每批最多 80000 字符处理全部有效阶段总结，所有批次成功后才原子替换旧骨架；重建不再使用旧骨架作为内容来源；
- 骨架每次维护携带最多 20000 字符蓝灯常驻世界书和最多 10000 字符本批绿灯命中条目；骨架只记录长期重要历史和大纲，当前角色状态与NPC档案交给MVU变量和世界书；
- 骨架默认最大 `5000 Token`，新增 `512–10000 Token` 配置项；主连接、自定义接口及空响应重试上限同步放宽到 10000；
- 设置页支持查看和编辑骨架，但不提供删除并拒绝空白或超限保存；人工版本作为后续增量更新的权威基线；
- 阶段总结编辑、删除、Swipe、删楼、分支变化或下调上限会使骨架过期；旧正文保留供核对和重建，但过期期间不注入请求；
- 最近请求 Token 卡片、状态栏、测试统计、检查器和调试报告已纳入骨架状态与用量。

## 0.19.3 阶段总结缺省标题修复（已由 0.20.2 自由格式取代）

- 当模型输出保留其余四个合法标题、仅漏掉一个位于中间的空分区时，本地按原顺序补齐该标题并写入“无”，避免整批长历史因常见格式小误差被丢弃。
- 修复范围保持严格：标题乱序、缺少多个分区、缺少首尾分区仍会拒绝写入，并在调试轨迹中记录是否执行过标题补齐。

## 0.19.2 长历史阶段总结修复

- 阶段总结单批来源上限提升到 64,000 字符，并允许在完整轮次边界关闭的限长批次立即生成总结，避免超长回复永久阻塞后续历史；
- “处理窗口外历史”按实际生成数量反馈结果，不再把零总结误报为处理成功；
- 扩展版本与静态资源查询参数同步升级，确保更新后浏览器加载修复后的构建而不是继续复用 `0.19.1` 缓存。

## 0.19.1 Token 卡片窄栏适配

- 标题、总 Token、模型/预设信息、StoryEcho 三项明细和完整分类名称均允许完整换行，移除会隐藏数据的省略号；
- 中等宽度使用两列 StoryEcho 明细与单列分类，极窄宽度进一步切为全单列布局。

## 0.19.0 最近请求 Token 构成

- 设置页新增最近一次已完成角色请求的 Token 卡片，显示总输入 Token、模型/预设/Tokenizer，以及系统预设、角色卡、世界书、示例、最近原文、StoryEcho 注入、其他扩展和未分类占比；
- StoryEcho 本轮发送另行汇总最近原文上下文、阶段总结和剧情元数据注入，元数据继续拆分为当前状态校正与动态召回；

- 只选择消息 ID 已落入当前聊天的最新提示词，避免停止生成后尚未成楼或后台 quiet 请求污染卡片；按聊天和提示词修订缓存计数，并在聊天加载、完成/停止生成、Swipe 和明细保存后自动刷新。

## 0.17.0 两层功能模式

- 设置页只保留“StoryEcho 上下文管理”和“剧情记忆与召回”两个功能开关；新安装默认只准备基础模式；
- 基础模式只依赖 LLM，自动执行最小原文窗口、独立阶段总结和安全裁剪，不读取 Embedding 或 Vector Storage；
- 剧情记忆开关开启后统一启用抽取、整理、向量同步、查询改写、向量检索和请求级召回，不再暴露自动抽取、自动总结或自动注入等重叠开关；
- 关闭剧情记忆时保留已有元数据、向量及待同步队列，仍允许查看、修改和删除；重建按钮被禁用，编辑/删除不会主动访问向量服务；
- 阶段总结拥有独立来源哈希和覆盖游标，记忆关闭时不再受 `indexedThroughMessageId` 限制；记忆开启时仍等待抽取覆盖，保证总结可使用同批结构化校正；
- 旧版设置会迁移到 7 版：已有安装按旧自动抽取或有效召回配置保持记忆模式，新安装默认关闭记忆层；
- 自动化回归覆盖新安装默认值、旧设置迁移、开关兼容字段、基础模式总结推进、禁止抽取/向量调用，以及请求中保留总结但不注入记忆。

## 0.16.0非阻塞前台与剧情阶段边界

- 生成前不再同步执行抽取、总结、向量写入或分支旧向量清理；后台未覆盖的原文继续保留，分支元数据先本地失效并由回复后后台清理向量；
- 明确结束上一阶段并开始新阶段时，默认隔离较早阶段的自动记忆与阶段总结，并在边界前提前封闭总结批次；篇章、任务、旅程、冒险、故事线等题材共用同一规则，普通换场与时间推进不触发隔离；置顶、人工编辑、明确点名且当前阶段没有同名实体的旧事实和直接较早阶段查询保留跨阶段能力；
- 证据边界审计进入严格事实模式；近期User纠正、总结否定项和非confirmed推断不会进入确定性当前状态账本；
- 修复复合地点值与“人物名 + 随身携带”被专名落地校验误拒的问题；阶段总结会保守删除同一输出中已经解决却仍标为未知的用途、位置、持有者、身份或真伪条目。

## 当前安全默认值

- 扩展默认关闭；
- LLM 默认使用主连接；
- 自定义 Key 明文保存在当前用户扩展设置中，以换取持久化和多端同步；
- Key 不进入聊天数据、Vector Storage 或调试报告；
- 自定义 LLM 由 SillyTavern Chat Completions 后端转发；自定义 Embedding 由内置 `/proxy/` 转发；
- 索引没有覆盖裁剪边界时保留完整聊天；
- 阶段总结没有覆盖裁剪边界时保留未总结原文；
- 正常生成不执行抽取、总结或待处理向量写入；
- 未凑满配置抽取轮数时不执行抽取，并扩大安全原文窗口；
- 后台未覆盖的索引或总结范围继续保留原文；
- 检索、抽取或自定义 Provider 失败时优先放行正常生成或按设置回退。
